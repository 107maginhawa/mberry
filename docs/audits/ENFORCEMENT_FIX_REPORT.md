## P0 — EU-CONTRAST-text-secondary-bg-white → REJECTED (false positive)

**Finding (rev 2):** `text-secondary` on `bg-white` = 1.07:1, fails WCAG AA. count=2.
**Suggested fix (rev 2):** swap `text-secondary` → `text-secondary-foreground`/`text-muted-foreground`.

**Verdict: FALSE POSITIVE. Suggested fix NOT applied** (it would recolor working tabs for zero a11y gain).

### Evidence

1. **No bare `text-secondary` exists in the codebase.**
   `grep -rn 'text-secondary' apps/*/src | grep -vE 'color-text-secondary|text-secondary-foreground'` → **zero matches.**
   Every `text-secondary` substring is either the arbitrary-value class `text-[var(--color-text-secondary)]` or `text-secondary-foreground`.

2. **The 2 flagged sites are both `<TabsTrigger>`** with identical classes:
   - `apps/memberry/src/features/membership/components/institutional-membership-table.tsx:98`
   - `apps/memberry/src/features/membership/components/member-table.tsx:203`
   - `className="text-[var(--color-text-secondary)] data-[state=active]:text-[var(--color-text)] data-[state=active]:bg-white"`
   - Parent `TabsList` bg = `--color-surface-warm` (NOT white).

3. **Real contrast passes AA.** `--color-text-secondary: #554B60` (globals.css:31).
   `#554B60` on white = **8.20:1** (WCAG AA needs 4.5:1). The audit's 1.20:1 came from mis-resolving to `--secondary` (#F0E8EC pink).

4. **Pink-on-white cannot render.** `bg-white` is gated behind `data-[state=active]:`, and the active state *also* recolors text to `--color-text` (dark). So when bg IS white, text is dark; when text is `#554B60`, bg is `--color-surface-warm`. The pink/white pair the audit reported never co-exists at render time.

### Root cause (two compounding tokenizer errors in the UI-consistency engine)

- **(a) Arbitrary-value-class conflation:** `text-[var(--color-text-secondary)]` tokenized as the bare `text-secondary` utility → mapped to `--secondary` (#F0E8EC) instead of `--color-text-secondary` (#554B60).
- **(b) State-variant blind pairing:** a `data-[state=active]:`-gated `bg-white` paired with the base-state text color, ignoring that the same variant overrides the text color.

### Action taken

Added `// oli-ui: exempt(reason=...)` annotation to both lines documenting the render-time-safe invariant. No className/render change. Logged the engine limitation to `CHECK_LEARNINGS.md` (category `low-confidence-heuristic`). Upstream engine fix filed separately by maintainer.
