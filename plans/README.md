  leak / concurrent-offer race"** (signaling-client.ts, peer-connection.ts):
  these files are **unfinished stubs** — the constructor `_token` is "kept for

  fail-safe — concurrent POSTs that both miss the cookie just both `GET

  implicit re-seed on retry. `seedCsrfToken` swallows failures by documented
  design. Not a bug. transport/csrf are clean.
- **SDK `use-auth` module-level hook cache / optimistic-mutation rollback edge
  cases**: the subagent cited impossible line numbers (`use-auth-hook.ts:901`,
  `file-upload.ts:1549` — those files are tens of lines), i.e. fabricated
