## Anti-Patterns Verdict
Not AI slop. Honest em-dash stats (no fake zeros) + careful auth gating (no protected-route flash) show real engineering thought. Closest tell: the 6-tile stat grid (hero-metric / identical-card-grid family), but restrained. Detector clean.

## What's Working
- **Honest data states.** `statsStatus` branches loading/error/ready, em-dash for missing, "No snapshot for this month yet" instead of confident zeros. Real money via `centavosToPhp`. Genuinely good system-status integrity.
- **All four UI states present** (skeleton / error role=alert / empty / table). DESIGN.md state vocabulary respected.
- **Careful auth gate** in `__root.tsx` — won't render a protected route for an unauthed user even one frame; explicit `forbidden` screen for non-operators.

## Priority Issues
- **[P1] No app shell.** RootGate renders bare `<Outlet/>`. No header, operator identity, or **sign-out** anywhere in the app. Org app adopted a shared shell+sign-out; console didn't. Operator is trapped signed-in. Fix: adopt shared shell/header with sign-out. Cmd: `/impeccable adapt`
- **[P2] Inconsistent design vocabulary within one app.** OrgsView uses raw Tailwind (`text-2xl`,`text-sm`,`font-bold`) + inline `style={{fontSize:'18px'}}`; sign-in/__root use tokens (`text-section`,`text-body`). Sign-in hand-rolls `<input className="border">` while CreateOrg uses `<Input>`. Status shown as plain `{org.status}` not `StatusBadge` (DESIGN.md: status=text+color, shared badge). Fix: one vocabulary — tokens + shared Input + StatusBadge. Cmd: `/impeccable polish`
- **[P2] Operator efficiency gaps.** Org table has no search/filter/sort; create form has no Cancel. A founder with 20+ orgs scrolls a flat table. Fix: add search + sortable columns; Cancel link on form. Cmd: `/impeccable shape`
- **[P3] Stats strip undifferentiated.** 6 identical tiles, no hierarchy; Revenue (money) not visually distinct nor `tabular-nums`. Fix: group/scale tiles, emphasize money. Cmd: `/impeccable layout`
- **[P3] Raw error strings + no retry.** Server `error` rendered verbatim; "Stats unavailable"/"Failed to load" have no retry affordance. Cmd: `/impeccable clarify`

## Persona Red Flags
**Alex (operator/power user):** No sign-out. No search/sort on org list. No keyboard shortcuts, no bulk. Flat table doesn't scale past a screenful.
**Riley (stress tester):** Long org names may overflow table cells; `createdAt` falls back to `String(...)` if not a Date; "No organizations yet" empty state has no inline CTA (create is only in header). Pagination drift (use-orgs total vs rendered rows) means the table may silently under-show.
**Sam (a11y):** Sign-in raw `<input>` relies on browser default focus ring (no `focus-visible` token); rest labeled + min-h-tap present.

## Minor Observations
- No page max-width: stats + table stretch edge-to-edge on wide monitors.
- `orgType`/`status` enum values shown raw lowercase.
- Create-org correctly a full route, not a modal (good).

## Questions to Consider

- At what org count does a flat no-search table break the founder's flow?
