# Plan 17-01: Admin SDK migration + error boundaries

## Objective

Replace all 11 raw fetch() calls in admin app routes with SDK-generated query hooks. Add ErrorBoundary to admin and account app root layouts.

## Tasks

<task id="1">
<title>Replace fetch calls in admin dashboard (index.tsx)</title>
<read_first>
- apps/admin/src/routes/index.tsx
- packages/sdk-ts/src/generated/@tanstack/react-query.gen.ts (search for admin-related query options — listAssociations, listOrganizations, etc.)
- apps/memberry/src/routes/_authenticated/dashboard.tsx (reference for SDK hook usage pattern)
</read_first>
<action>
In `apps/admin/src/routes/index.tsx`, find the `useQuery` calls that use inline `queryFn: async () => fetch('/api/admin/...')` pattern. Replace each with the corresponding SDK-generated query options.

The SDK generates options functions like `listAssociationsOptions()`, `listOrganizationsOptions()`, etc. Import them from `@monobase/sdk-ts/generated/react-query`.

Pattern to follow (from memberry app):
```tsx
import { listAssociationsOptions } from '@monobase/sdk-ts/generated/react-query';

// Replace:
const { data } = useQuery({ queryKey: ['associations'], queryFn: () => fetch('/api/admin/associations').then(r => r.json()) });
// With:
const { data } = useQuery(listAssociationsOptions());
```

If admin endpoints don't have matching SDK hooks (because they're under `/admin/*` which uses different auth), keep the fetch pattern but wrap in proper error handling. Check what SDK hooks are available for admin routes.
</action>
<acceptance_criteria>
- index.tsx uses SDK query options where available
- No raw `fetch('/api/admin/...')` calls remain where SDK hooks exist
- If SDK hooks don't cover admin endpoints, document which ones still use fetch
</acceptance_criteria>
</task>

<action>
Add ErrorBoundary component to both root routes. Check how memberry implements it and replicate.

If memberry uses a class-based ErrorBoundary:
```tsx
class ErrorBoundary extends React.Component<{ children: React.ReactNode }, { hasError: boolean; error?: Error }> {
  state = { hasError: false, error: undefined as Error | undefined };
  static getDerivedStateFromError(error: Error) { return { hasError: true, error }; }
  render() {
    if (this.state.hasError) {
      return (
        <div className="flex flex-col items-center justify-center min-h-screen p-4">
          <h1 className="text-2xl font-bold mb-4">Something went wrong</h1>
          <p className="text-muted-foreground mb-4">{this.state.error?.message}</p>
          <button onClick={() => window.location.reload()} className="px-4 py-2 bg-primary text-primary-foreground rounded">
            Reload page
          </button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

Or if TanStack Router has built-in errorComponent support, use that:
```tsx
export const Route = createRootRoute({
  errorComponent: ({ error }) => (
    <div>Something went wrong: {error.message}</div>
  ),
  // ...existing config
});
```

Check which pattern memberry uses and follow it.
</action>
