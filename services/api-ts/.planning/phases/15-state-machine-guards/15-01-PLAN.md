# Plan 15-01: Election status transition guards

## Objective

Add transition validation to `updateElectionStatus.ts` which currently accepts ANY status with zero guards. The election has 6 statuses: draft, nominationsOpen, votingOpen, awaitingConfirmation, published, cancelled.

</read_first>
<action>
In `updateElectionStatus.ts`, add a transition map before the update call:

```typescript
const VALID_TRANSITIONS: Record<string, string[]> = {
  draft: ['nominationsOpen', 'cancelled'],
  nominationsOpen: ['votingOpen', 'cancelled'],
  votingOpen: ['awaitingConfirmation', 'cancelled'],
  awaitingConfirmation: ['published', 'cancelled'],
  published: [],  // terminal state
  cancelled: [],  // terminal state
};

// Fetch current election
const existing = await repo.findById(id);
if (!existing) throw new NotFoundError('Election not found');

const allowed = VALID_TRANSITIONS[existing.status] || [];
if (!allowed.includes(body.status)) {
  throw new BusinessLogicError(
    `Cannot transition election from '${existing.status}' to '${body.status}'. Allowed: ${allowed.join(', ') || 'none (terminal state)'}`,
    'INVALID_ELECTION_TRANSITION'
  );
}
```

Import BusinessLogicError from the error utilities (check how publishAnnouncement.ts imports it).
