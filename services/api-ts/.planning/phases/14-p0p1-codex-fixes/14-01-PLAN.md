# Plan 14-01: Fix org-context middleware bugs + public directory allowlist

## Objective

Fix 3 Codex-verified bugs: (1) UUID regex path fallback misidentifies resourceId as orgId, (2) body-supplied organizationId ignored by middleware, (3) public directory allowlist mismatch.

## Tasks

<task id="1">
<title>Remove UUID regex fallback from org-context middleware</title>
<read_first>
- services/api-ts/src/middleware/org-context.ts
- services/api-ts/src/middleware/org-context.test.ts
</read_first>
<action>
In `org-context.ts`, remove lines 53-55 (the `uuidInPath` regex) and remove `uuidInPath?.[1]` from the orgId resolution chain at line 61. The orgId should only come from:
1. `x-org-id` header
2. `orgId` query param
3. `organizationId` query param
4. `organizationId` path param

Remove the regex: `const uuidInPath = ctx.req.path.match(/\/([0-9a-f]{8}-...)/i);`
Remove from chain: `uuidInPath?.[1] ??`
</action>
<acceptance_criteria>
- org-context.ts does NOT contain `uuidInPath`
- org-context.ts does NOT contain regex pattern `[0-9a-f]{8}`
- orgId resolution uses only header, query, and path param sources
</acceptance_criteria>
</task>

<task id="2">
<title>Add request body extraction for organizationId</title>
<read_first>
- services/api-ts/src/middleware/org-context.ts
</read_first>
<action>
After the header/query/param extraction chain, add body extraction as a fallback for POST/PUT/PATCH methods only:

```typescript
let orgId =
  ctx.req.header('x-org-id') ??
  ctx.req.query('orgId') ??
  ctx.req.query('organizationId') ??
  ctx.req.param('organizationId') ??
  null;

// Fallback: extract from request body for mutation methods
if (!orgId && ['POST', 'PUT', 'PATCH'].includes(ctx.req.method)) {
  try {
    const body = await ctx.req.json();
    orgId = body?.organizationId ?? body?.orgId ?? null;
  } catch {
    // Body not JSON or already consumed â€” continue without
  }
}
```
</action>
<acceptance_criteria>
- org-context.ts contains `await ctx.req.json()` in a try/catch block
- Body extraction only runs for POST, PUT, PATCH methods
- Body extraction is the last fallback (after header, query, path param)
</acceptance_criteria>
</task>
