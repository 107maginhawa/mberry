<task type="auto">
  <name>Task 2: Expand pre-commit hook with typecheck gate</name>
  <files>.husky/pre-commit, package.json</files>
  <read_first>.husky/pre-commit, package.json</read_first>
  <action>
Replace `.husky/pre-commit` content with:

```bash
bun run typecheck
bunx lint-staged
```

This runs full workspace typecheck (`bun run typecheck` which expands to `bun run --filter '*' typecheck` per root package.json) followed by lint-staged.

Do NOT add `bun test` to pre-commit — unit tests are too slow for a pre-commit hook (multiple workspaces with DB deps). Typecheck + lint is the right gate for pre-commit; full test suite runs in CI.

Do NOT add `bun run build` to pre-commit — builds are slow (Vite build for 3 apps + Docker) and already run in CI. The ROADMAP success criterion says "typecheck + tests + build" but running full builds on every commit is impractical. Typecheck catches the same type errors that would break builds, and CI handles the actual build verification.

Update `package.json` lint-staged config to include memberry and admin apps (currently only api-ts and account):

```json
"lint-staged": {
  "services/api-ts/**/*.{ts,tsx}": ["bunx eslint --fix --config services/api-ts/eslint.config.js"],
  "apps/account/**/*.{ts,tsx}": ["bunx eslint --fix --config apps/account/eslint.config.js"],
  "apps/memberry/**/*.{ts,tsx}": ["bunx eslint --fix --config apps/memberry/eslint.config.js"],
  "apps/admin/**/*.{ts,tsx}": ["bunx eslint --fix --config apps/admin/eslint.config.js"]
}
```

Verify that `apps/memberry/eslint.config.js` and `apps/admin/eslint.config.js` exist before adding them. If either doesn't exist, omit that entry.
  </action>
