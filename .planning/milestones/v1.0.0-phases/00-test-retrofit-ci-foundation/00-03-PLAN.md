Per D-07 (Husky + lint-staged for typecheck + lint only, ~15-30s). Per D-08 (existing /pre-commit skill stays as manual comprehensive check).

1. Install dependencies:
```bash
cd /Users/elad-mini/Desktop/memberry
bun add -d husky lint-staged
```

2. Initialize Husky:
```bash
bunx husky init
```
This creates `.husky/` directory. If `husky init` doesn't work with bun, manually:
```bash
mkdir -p .husky
```

3. Create `.husky/pre-commit`:
```sh
#!/usr/bin/env sh
. "$(dirname -- "$0")/_/husky.sh"

bunx lint-staged
```

Make it executable: `chmod +x .husky/pre-commit`

4. Add to root `package.json`:

Add `"prepare": "husky"` to scripts section.

Add `lint-staged` config at top level:
```json
{
  "lint-staged": {
    "*.{ts,tsx}": [
      "bunx eslint --fix --max-warnings=0"
    ]
  }
}
```

Note on typecheck: `tsc --noEmit` cannot run on individual staged files (it needs the full project). Two options:
- Option A: Run `bunx tsc --noEmit` on the whole project in lint-staged (slower, ~10-15s but still under 30s threshold)
- Option B: Only run eslint in lint-staged, rely on CI for typecheck

Choose Option A for safety (per D-07 which specifies "typecheck + lint"). Add as a separate entry:
```json
{
  "lint-staged": {
    "*.{ts,tsx}": [
      "bunx eslint --fix --max-warnings=0"
    ],
    "**/*.ts?(x)": [
      "bash -c 'bunx tsc --noEmit'"
    ]
  }
}
```

Actually, lint-staged runs commands per-file by default. For tsc which needs full project context, use the bash -c wrapper pattern so it runs once regardless of how many files are staged.

5. Ensure root package.json has `typecheck` and `lint` scripts if not already present:
- `"typecheck": "bunx tsc --noEmit"` (or `bun --filter '*' run typecheck` if workspaces have individual typecheck scripts)
- `"lint": "bunx eslint ."` (or equivalent)

These are needed by ci.yml (Plan 02) as well.
