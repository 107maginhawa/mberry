**Account app migration:**

1. Add `"@monobase/ui": "workspace:*"` to `apps/account/package.json` dependencies.

2. Update `apps/account/tailwind.config.ts` content array — add `"../../packages/ui/src/**/*.{ts,tsx}"` entry.

3. Replace `apps/account/src/lib/utils.ts` with a re-export shim:
   ```typescript
   export { cn } from "@monobase/ui/lib/utils"
   ```
   This keeps all existing `@/lib/utils` imports working without mass find-replace.

4. For each component that exists in BOTH `apps/account/src/components/` AND `packages/ui/src/components/`, update the import path in all consuming files. Use this approach:
   - Run `grep -rl 'from "@/components/button"' apps/account/src/` to find all consumers
   - Replace `from "@/components/button"` with `from "@monobase/ui/components/button"` (or `from "@monobase/ui"` for barrel imports)
   - Repeat for each shared component: card, badge, input, label, select, dialog, dropdown-menu, table, tabs, form, skeleton, avatar, checkbox, switch, sheet, separator, scroll-area, progress, textarea, tooltip, alert, popover, alert-dialog, command, calendar, slider, toggle, toggle-group

5. Delete the now-unused local copies from `apps/account/src/components/` — ONLY the ones that were extracted to packages/ui. Keep app-specific components: app-sidebar.tsx, sidebar.tsx, sonner.tsx, loading.tsx, logo.tsx, not-found.tsx, phone-input.tsx, image-cropper-dialog.tsx, datetime-filter.tsx, combobox.tsx, pagination.tsx

6. Run `cd /Users/elad-mini/Desktop/memberry && bun install` to link workspace dep.

7. Run `cd apps/account && bunx tsc --noEmit` — fix any import errors.

**Memberry app migration:**

8. Add `"@monobase/ui": "workspace:*"` to `apps/memberry/package.json` dependencies.
