  <action>
Add two new jobs to the existing ci.yml workflow:

**Job: build-api**
- runs-on: ubuntu-latest
- timeout-minutes: 15
- Steps:
  1. actions/checkout@v4
  2. Set up Docker Buildx (docker/setup-buildx-action@v3)
  3. Build API Docker image (docker/build-push-action@v6):
     - context: `.` (monorepo root — Dockerfile expects this)
     - file: `services/api-ts/Dockerfile`
     - push: false (PR builds don't push)
     - tags: `ghcr.io/${{ github.repository }}/api:${{ github.sha }}`
     - cache-from: type=gha
     - cache-to: type=gha,mode=max
     - load: true (so image is available locally)
  4. Verify image health: `docker run --rm -d --name api-health-test -p 7213:7213 -e DATABASE_URL=postgresql://fake -e AUTH_SECRET=test ghcr.io/${{ github.repository }}/api:${{ github.sha }} && sleep 2 && docker stop api-health-test` (just verifies image starts without crash — no DB available so /readyz will fail, but image should load)

**Job: build-frontends**
- runs-on: ubuntu-latest
- timeout-minutes: 10
- Steps:
  1. actions/checkout@v4
  2. oven-sh/setup-bun@v2 with bun-version: 1.2.21
  3. bun install --frozen-lockfile
  4. Build OpenAPI spec: `bun --filter @monobase/api-spec run build`
  5. Generate API codegen: `bun --filter @monobase/api-ts run generate`
  6. Build each frontend app:
     ```
     cd apps/memberry && bun run build
     cd ../admin && bun run build
     cd ../account && bun run build
     ```
     Set env: `VITE_API_URL: ""` (empty string — will be set per-environment at deploy time)
  7. Upload each app's dist/ as artifact using actions/upload-artifact@v4:
     - name: frontend-memberry, path: apps/memberry/dist/
     - name: frontend-admin, path: apps/admin/dist/
     - name: frontend-account, path: apps/account/dist/

Both jobs run in parallel with the existing lint-typecheck, e2e, and contract jobs (no `needs` dependency). They validate that builds succeed on every PR.

Do NOT modify the existing lint-typecheck, e2e, or contract jobs in any way.
  </action>
