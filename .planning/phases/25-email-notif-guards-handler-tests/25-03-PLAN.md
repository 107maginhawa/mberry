<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet -> unsubscribe endpoint | Unauthenticated users submit token; must be HMAC-verified |
| Officer -> suppression list | Must be org-scoped; cross-org reads blocked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-07 | Tampering | unsubscribeEmail.ts | mitigate | HMAC-SHA256 token verification before any DB write; reject invalid tokens with 400 |
| T-25-08 | Denial of Service | unsubscribeEmail.ts | accept | Endpoint is simple verify+write; no expensive operations; standard rate limiting via reverse proxy |
| T-25-09 | Elevation of Privilege | listEmailSuppressions.ts | mitigate | Require admin role; scope all queries by organizationId from session context |
| T-25-10 | Spoofing | unsubscribeEmail.ts | mitigate | Token is HMAC(email+orgId) — cannot be enumerated without server secret |
</threat_model>
