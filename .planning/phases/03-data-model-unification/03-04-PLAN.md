<objective>

Root cause: Tests pass `{ organizationId: null }` to makeCtx() expecting a 403, but handlers check `ctx.get('orgId')`. The null organizationId never reaches orgId (defaults to 'org-1'), so the guard never fires.

Fix: When `organizationId` is explicitly passed in overrides, use it as `orgId` too (unless orgId is also explicitly passed).
</objective>
