<objective>
Add Zod input validation to updateMember handler (V-20).

Currently `updateMember.ts` accepts `body.status` as a raw string from `ctx.req.json()` with no validation. While `isValidTransition()` silently rejects invalid transitions, the handler still processes the request and returns 200 even with garbage status values like "banana". This should return 400.

TDD approach:
1. RED: Write tests asserting 400 for invalid status values
2. GREEN: Add Zod schema to updateMember that validates status against the allowed enum
3. REFACTOR: Ensure no duplication between Zod schema and VALID_TRANSITIONS map
</objective>

   - Test: `body.status = "active"` → 200 (valid, no-op transition)
