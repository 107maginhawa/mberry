<objective>
Add unique constraint on (election_id, voter_id, position_id) to the election votes table to prevent duplicate votes at the DB level.
</objective>

<context>
See 15-CONTEXT.md for key business rules and file inventory.
See 15-PATTERNS.md for test infrastructure and transaction patterns.

Bug: `castBallot.ts` has an app-level duplicate vote check, but no DB unique constraint. Concurrent requests can bypass the app-level check and create duplicate votes. Plan 15-09 adds the app-level check; this plan adds the DB backstop.
</context>

<tasks>
## Data audit
1. Query existing vote data for duplicate (electionId, voterId, positionId) tuples
2. Document any duplicates and decide cleanup strategy

## Migration
3. Write data cleanup migration that deduplicates any existing vote records
4. Add unique index to the election votes schema:
   ```typescript
   .unique('election_vote_unique', ['electionId', 'voterId', 'positionId'])
   ```
5. Run `bun run db:generate` to produce the migration SQL

## Verification
6. Apply migration to dev DB
7. Attempt to insert duplicate vote — verify DB rejects with unique violation
8. Verify 2123+ tests still pass
</tasks>
