<objective>
Replace hardcoded `billingCycle: 'annual'` in settle-payment.ts and markDuesInvoicePaid.ts with the actual billingFrequency from the org's dues config. Support quarterly, semi-annual, and annual cycles.
</objective>

<context>
See [15-CONTEXT.md](./15-CONTEXT.md) and [15-PATTERNS.md](./15-PATTERNS.md) for domain remediation background.

**Current bug**: 
- `settle-payment.ts:81` hardcodes `billingCycle: 'annual'`
- `markDuesInvoicePaid.ts:52` hardcodes `billingCycle: 'annual'`
- The dues config schema already has a `billingFrequency` field, but it's never read or consumed by the settlement logic
- Organizations configured for quarterly or semi-annual billing still get 12-month expiry extensions
</context>

## RED phase — write failing tests first

1. Read `settle-payment.ts` to find the hardcoded `billingCycle: 'annual'` at line 81
2. Read `markDuesInvoicePaid.ts` to find the hardcoded `billingCycle: 'annual'` at line 52
3. Read the dues config schema to understand the `billingFrequency` field type and values
4. Read `getDuesConfig.ts` to understand how dues config is fetched
5. Create or extend `settle-payment.test.ts` with test cases:
   - **Quarterly billing extends by 3 months**: Org config has `billingFrequency: 'quarterly'`, verify expiry extended by 3 months
   - **Semi-annual billing extends by 6 months**: Org config has `billingFrequency: 'semi-annual'`, verify expiry extended by 6 months
   - **Annual billing extends by 12 months**: Org config has `billingFrequency: 'annual'`, verify expiry extended by 12 months (current behavior)
   - **Missing config defaults to annual**: No dues config found for org, verify expiry extended by 12 months
   - **Null billingFrequency defaults to annual**: Config exists but `billingFrequency` is null, defaults to 12 months
6. Run tests — confirm quarterly and semi-annual tests fail

## GREEN phase — consume billingFrequency

7. Modify `settle-payment.ts`:
   - Fetch the org's dues config (or accept it as a parameter)
   - Read `billingFrequency` from the config
   - Map frequency to months: `quarterly` → 3, `semi-annual` → 6, `annual` → 12
   - Pass the computed months to `computeNewExpiry` instead of hardcoded annual
   - Default to `annual` (12 months) if config is missing or `billingFrequency` is null
8. Modify `markDuesInvoicePaid.ts`:
   - Apply the same pattern: read billingFrequency, compute months, pass to expiry logic
   - Or delegate to settle-payment if it already handles the full flow
9. Run tests — confirm all pass

## REFACTOR phase
