## Task 1: Create dunning schema

File: `services/api-ts/src/handlers/association:member/repos/dunning.schema.ts` (NEW)

Tables:
1. `dunningTemplates` pgTable:
   - `id` uuid primary key
   - `organizationId` uuid NOT NULL
   - `name` varchar(100) NOT NULL
   - `stage` integer NOT NULL (1-5)
   - `daysAfterDue` integer NOT NULL (>=0)
   - `channel` varchar(20) NOT NULL — enum: email, sms, in-app
   - `subject` varchar(200) — nullable, for email
   - `body` text NOT NULL
   - `status` varchar(20) NOT NULL default 'active' — active/inactive
   - Plus baseEntityFields

2. `dunningEvents` pgTable:
   - `id` uuid primary key
   - `membershipId` uuid NOT NULL
   - `personId` uuid NOT NULL
   - `templateId` uuid NOT NULL (FK to dunningTemplates)
   - `stage` integer NOT NULL
   - `sentAt` timestamp NOT NULL
   - `channel` varchar(20) NOT NULL
   - `deliveryStatus` varchar(20) NOT NULL default 'pending'
   - Plus baseEntityFields
