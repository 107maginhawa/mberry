Per D-01 and D-04: Add a new `coverage-gate` job to `.github/workflows/contract.yml`. This job runs INDEPENDENTLY of the `contract` job (no `needs:` dependency) since it is pure static analysis.

Add the following job after the existing `contract` job:

```yaml
  coverage-gate:
    runs-on: ubuntu-latest
    timeout-minutes: 5

    steps:
      - uses: actions/checkout@v4

      - name: Install Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: 1.2.21

      - name: Install workspace deps
        run: bun install --frozen-lockfile

      - name: Test registry coverage
        run: bun run test:registry

      - name: BR coverage gate
        run: bun run test:br --ci
```

Key design points per D-04:
- NO `services:` block (no Postgres, no MinIO) — pure static analysis
- NO `needs: [contract]` — runs in parallel with contract tests for faster CI
- `timeout-minutes: 5` — these scripts complete in seconds
- Two separate steps so failures are clearly attributable
- `bun run test:br --ci` uses the `--ci` flag which exits non-zero on regressions (per CONTEXT D-03)
