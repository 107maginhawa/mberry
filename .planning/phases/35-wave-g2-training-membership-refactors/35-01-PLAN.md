<objective>
Fix credit cycle start date in markComplete handler (V-12).

`markComplete.ts:49` calls `getCycleForDate(activityDate, activityDate, 2)` — using the training activity date as BOTH the registration anchor and target date. This means every credit entry starts a new cycle anchored to the activity, instead of using the member's actual registration date.

The `getCycleForDate` function signature is `(registrationDate, targetDate, cyclePeriodYears)`. The first param should be the member's membership start date (registration), not the activity date. The second param (target) is correctly the activity date.

Fix: Look up the member's membership record to get their start/registration date, pass that as first arg.
</objective>
