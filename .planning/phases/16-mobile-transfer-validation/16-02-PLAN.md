# Plan 16-02: Mobile Viewport E2E Tests

## Goal
Validate critical user flows render and function correctly at mobile viewport (375×812, iPhone SE/13 mini). Add mobile device profile to Playwright config.

## Context
- Playwright config (`apps/memberry/playwright.config.ts`) has only Desktop Chrome
- Mobile nav components exist: `officer-mobile-nav.tsx`, `member-bottom-nav.tsx`
- No mobile viewport tests exist across any app
- Target viewport: 375×812 (standard iPhone)

## Tasks

### Task 1: Add mobile project to Playwright config
- File: `apps/memberry/playwright.config.ts`
- Add `iPhone 13` device project (375×812 viewport, touch enabled)
- Keep desktop project as default; mobile as separate named project

### Task 2: Create mobile viewport test file
- File: `apps/memberry/tests/e2e/mobile/viewport.spec.ts`
- Configure tests to run only on mobile project

### Task 3: RED — Dashboard mobile layout
```
Test: "dashboard renders without horizontal scroll at 375px"
- Sign in as member
- Navigate to dashboard
- Verify no horizontal overflow (scrollWidth <= clientWidth)
- Verify key dashboard cards are visible and stacked vertically
```

### Task 4: RED — Officer mobile navigation
```
Test: "officer mobile nav shows hamburger and drawer"
- Sign in as officer
- Navigate to officer area
- Verify desktop sidebar is hidden at 375px
- Verify mobile nav header (48px sticky) is visible
- Open drawer menu, verify nav links visible
- Click a nav item, verify navigation works
```

### Task 5: RED — Payment form mobile
```
Test: "dues payment form is usable at mobile width"
- Sign in as treasurer
- Navigate to dues page
- Verify form fields don't overflow viewport
- Verify submit button is visible without scrolling past the fold
```

### Task 6: RED — Event registration mobile
```
Test: "event list and registration work at mobile width"
- Sign in as member
- Navigate to events page
- Verify event cards stack vertically
- Verify no horizontal overflow
```

### Task 7: GREEN — Fix layout issues found
- Fix any CSS/layout bugs that cause mobile viewport failures
- Common fixes: overflow-x hidden, flex-wrap, min-width removal

## Verification
- All mobile tests pass: `cd apps/memberry && bunx playwright test --project=mobile`
- Desktop tests still pass: `cd apps/memberry && bunx playwright test --project=chromium`
- No regressions
