**Test group 1: "Audit event capture via API"** (3 tests -- one per CRUD operation)

These tests sign in as admin, perform a write operation via API, then query GET /audit/logs to verify an audit event was captured.

Test 1: **"create operation produces audit event"** (cross-module: associations)
1. `await signInAsAdmin(page.context())`
2. POST to create an association: `context.request.post(\`${API_URL}/admin/associations\`, { data: { name: 'Audit Test Assoc ' + Date.now(), organizationId: '<org ID from seed>' } })`
3. Wait briefly (500ms) for fire-and-forget audit to complete
4. Query: `context.request.get(\`${API_URL}/audit/logs?action=create&limit=5\`)`
5. Parse response JSON, verify `data` array length > 0
6. Verify first entry has `action === 'create'`

Test 2: **"update operation produces audit event"** (cross-module: organizations)
1. Sign in as admin
2. PATCH to update an existing organization: `context.request.patch(\`${API_URL}/admin/organizations/<org-id>\`, { data: { name: 'Updated Org ' + Date.now() } })`
3. Wait 500ms
4. Query audit logs filtered by `action=update`
5. Verify an update audit event exists

Test 3: **"delete operation produces audit event"** (cross-module: feature-flags)
1. Sign in as admin
2. Create a feature flag first, then DELETE it: `context.request.delete(\`${API_URL}/admin/feature-flags/<flag-key>\`)`
3. Wait 500ms
4. Query audit logs filtered by `action=delete`
5. Verify a delete audit event exists

**Important notes for all API tests:**
- Use `page.context().request` for API calls (shares auth cookies)
- Use `signInAsAdmin(page.context())` before API calls
- Add `{ timeout: 10000 }` to expect assertions that depend on async audit writes
- Tests use 3 different admin-accessible resource types (associations, organizations, feature-flags) to verify cross-module audit coverage per CONTEXT.md locked decision

**Test group 2: "Audit dashboard"** (2 tests)

Test 4: **"audit page renders table with data"**
1. `await signInAndNavigate(page, '/audit')`
2. Wait for page to load: `await page.waitForSelector('table')`
3. Verify table headers exist: `await expect(page.locator('th:has-text("Action")')).toBeVisible()`
4. Verify at least one table row exists: `await expect(page.locator('tbody tr').first()).toBeVisible()`
5. Verify the page title "Audit Log" is visible

Test 5: **"audit page has filter controls"**
1. `await signInAndNavigate(page, '/audit')`
2. Verify filter controls exist:
   - `await expect(page.locator('select').first()).toBeVisible()` (action dropdown)
   - `await expect(page.locator('input[type="date"]').first()).toBeVisible()` (date filter)

**Cleanup:** Tests should not require cleanup -- feature flags and audit logs don't affect other tests. Use unique names with `Date.now()` suffix to avoid collisions.
