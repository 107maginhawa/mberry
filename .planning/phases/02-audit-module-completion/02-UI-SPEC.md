## Color

All values are CSS custom properties defined in `apps/admin/src/styles/globals.css`.

| Role | CSS Variable | Approx Hex | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` (hsl 0 0% 100%) | #ffffff | Main content area bg |
| Secondary (30%) | `--sidebar-background` (hsl 270 16% 18%) | #2D2635 | Sidebar (already implemented in __root.tsx) |
| Secondary surface | `--card` (hsl 0 0% 100%) | #ffffff | Table container bg (rounded-lg border bg-card) |
| Accent (10%) | `--sidebar-primary` (hsl 270 60% 65%) | ~#9B72CF | Active nav item indicator only |
| Muted | `--muted` / `--muted-foreground` | gray scale | Table headers, secondary text, empty states |
| Destructive | `--destructive` (hsl 0 84.2% 60.2%) | #ef4444 | Not used in Phase 2 (read-only audit dashboard) |

Accent reserved for:
- Active sidebar nav item background (`bg-white/10 text-white` per __root.tsx pattern)
- Audit log `Shield` icon color in sidebar nav link (same as other icons)

Do NOT use accent for: filter buttons, pagination controls, table row hover. Use `hover:bg-white/10` in sidebar and `bg-muted`/`text-muted-foreground` in content area.

Status badge colors (matches members page pattern):
- `success` outcome: `bg-green-500/10 text-green-600`
- `failure` outcome: `bg-red-500/10 text-red-600`
- Pending / unknown: `bg-muted text-muted-foreground`

Source: `globals.css` CSS vars + `members/index.tsx` badge pattern.

## Audit Dashboard Layout Spec

Route: `apps/admin/src/routes/audit/index.tsx`

```
┌─────────────────────────────────────────────────────┐
│ p-8 wrapper                                         │
│                                                     │
│  [Shield icon] Audit Log              [↻ Refresh]   │
│  View all write operations across all modules       │
│                                                     │
│  ┌ Filter Bar ────────────────────────────────────┐ │
│  │ Action [select▼]  Resource [text]              │ │
│  │ From [date]  To [date]  User [text]            │ │
│  └────────────────────────────────────────────────┘ │
│                                                     │
│  N events found                    Page 1 of X      │
│                                                     │
│  ┌ Table (rounded-lg border bg-card) ─────────────┐ │
│  │ Timestamp | Action | Resource | User | Outcome  │ │
│  │ ──────────────────────────────────────────────  │ │
│  │ row data...                                     │ │
│  └────────────────────────────────────────────────┘ │
│                                                     │
│  [← Prev]  Page 1 of X  [Next →]                   │
└─────────────────────────────────────────────────────┘
```

Table columns:
1. **Timestamp** — ISO date formatted as `YYYY-MM-DD HH:mm` (local), text-sm text-muted-foreground
2. **Action** — `create` / `update` / `delete`, badge with muted bg
3. **Resource Type** — string (e.g., `persons`, `membership`), text-sm
4. **Resource ID** — truncated to 8 chars with ellipsis, text-sm font-mono text-muted-foreground
5. **User** — user ID or "system" if unauthenticated, text-sm
6. **Outcome** — `success` (green badge) / `failure` (red badge)

Filter inputs (all optional, all trigger re-query via queryKey):
- `action`: `<select>` with options: All, create, update, delete
- `resourceType`: `<input type="text" placeholder="e.g. persons">` 
- `startDate` / `endDate`: `<input type="date">`
- `user`: `<input type="text" placeholder="User ID">`

Pagination: `limit=25` (default), `offset` increments by 25. Display `[← Prev]` / `[Next →]` as plain text buttons with border when active, disabled+muted when at boundary.

Refresh: single `<button>` with `RefreshCw` icon, calls `queryClient.invalidateQueries(['admin', 'audit-logs'])`. No auto-refresh (deferred per CONTEXT.md).

## States Required

Every data-fetching component must handle all 4 states:

| State | Trigger | UI |
|-------|---------|-----|
| Loading | `isLoading: true` | Single table row spanning all columns: "Loading audit events..." centered, text-muted-foreground |
| Empty (no data) | `data.length === 0`, no filters | Single table row: "No audit events recorded yet. Write operations will appear here automatically." |
| Empty (filtered) | `data.length === 0`, filters active | Single table row: "No events match your filters. Try adjusting the date range or clearing filters." |
| Error | `isError: true` | `<p className="text-sm text-red-500 mb-4">` above table (matches members page error pattern) |
| Populated | `data.length > 0` | Full table with rows + pagination |
