Create `services/api-ts/src/middleware/audit.ts` exporting `createAuditMiddleware()`:

1. The function returns an async Hono middleware `(ctx, next) => { ... }`
2. Call `await next()` FIRST (after-middleware pattern -- business logic runs before audit)
3. After next() returns, check `ctx.req.method` -- skip if not POST/PUT/PATCH/DELETE (return early for GET/HEAD/OPTIONS)
4. Get `audit` from `ctx.get('audit')` -- if undefined, return silently (guard)
5. Get `logger` from `ctx.get('logger')` and `user` from `ctx.get('user')`
6. Extract resource info from URL path:
   - `const url = new URL(ctx.req.url)`
   - `const segments = url.pathname.split('/').filter(Boolean)`
   - `resourceType = segments[0] ?? 'unknown'`
   - `resourceId = segments[1] ?? 'unknown'`
7. Map HTTP method to audit action: `{ POST: 'create', PUT: 'update', PATCH: 'update', DELETE: 'delete' }` -- type as `'create' | 'update' | 'delete'`
8. Derive outcome from `ctx.res.status`: `status >= 200 && status < 300 ? 'success' : 'failure'` -- type as `'success' | 'failure'`
9. Call `audit.logEvent(...)` with:
   - `eventType: 'data-modification'`
   - `category: 'association'`
   - `action` (from step 7)
   - `outcome` (from step 8)
   - `user: user?.id`
   - `userType: user ? 'client' : 'system'` (typed as `'client' | 'system'`)
   - `resourceType` (from step 6)
   - `resource: resourceId` (from step 6)
   - `description: \`${method} ${url.pathname}\``
   - `ipAddress: ctx.req.header('x-forwarded-for') || ctx.req.header('x-real-ip')`
   - `userAgent: ctx.req.header('user-agent')`
10. Wrap the entire audit.logEvent call in try/catch -- on error, call `logger?.error({ error }, 'Audit middleware failed to log event')` -- NEVER rethrow
